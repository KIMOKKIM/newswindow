# 기사·광고 저장이 재배포/재시작 후 사라지는 원인

## 코드 기준 저장 위치

| 데이터 | 기본 파일 | 환경 변수 오버라이드 |
|--------|-----------|----------------------|
| 광고 | `backend/data/ads.json` | `NW_ADS_JSON_PATH` 절대 경로 |
| 기사 | `backend/data/articles.json` | `NW_ARTICLES_JSON_PATH` 절대 경로 |

## 직접 원인

1. **호스트 파일시스템이 에페메럴** 인 경우(디스크 미부착 Render Web 등), 배포 산출물과 함께 컨테이너가 갈아끼워지며 **위 경로의 JSON이 초기화**될 수 있음.
2. **`NW_ADS_JSON_PATH` / `NW_ARTICLES_JSON_PATH` 가 운영에서 비어 있으면** 항상 기본 상대 경로를 쓰므로, **영구 볼륨을 쓰지 않는 한** 재시작마다 유실 위험이 반복됨.
3. **`loadAds()` 가 파일 없을 때 `getDefaultAds()`** 를 쓰는 것은 **조회 시 기본값 반환**이며, 파일이 있는 한 PUT으로 저장된 내용을 덮어쓰지 않음. 유실의 주원인은 **디스크 초기화**이지 seed가 성공 본문을 지우는 것이 아님.

## Free 플랜·디스크

- Render **Persistent Disk** 사용 가능 여부·크기·요금은 **Render 현행 정책**(대시보드)을 따른다.
- Free만으로 디스크를 붙이기 어렵다면 **외부 DB/객체 스토리지** 또는 **유료 인스턴스 + 디스크**가 필요하다.
