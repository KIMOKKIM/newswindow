# 헤더·사이드 광고·전체 구성 비교 리포트

## 비교 대상
- **참고(왼쪽 스크린)**: 대한뉴스형 포털 — 로고 좌우 **가로형 배너 2개**, 좌측 **세로 스택 4칸**, 우측 **세로 스택 3칸**(상단 스카이스크래퍼 + 하단 소형 2개), 바깥 배경 회색·중앙 화이트 박스.
- **기존(두 번째 스크린)**: 뉴스의창 구현 — 로고 좌우 **소형 배너(약 180×60 한도)**, 좌·우 **단일 윙 배너**만, 이미지 URL 비어 있을 때 **깨진 이미지 아이콘**, 헤더 슬롯이 시각적으로 **4칸처럼 보이는 경우**(빈 회색 칸) 등 포털형과 차이.

## 주요 차이 요약

| 항목 | 참고 페이지 | 기존 코딩 페이지 |
|------|-------------|------------------|
| **헤더 광고** | 로고 양옆 **넓은 가로 배너 2개** (대략 200~280×60~80px급, 가로 비중 큼) | `max-width:200px` 등으로 **상대적으로 작음** |
| **좌측 광고** | **4개 세로 적층** (중간은 4:3 느낌, 하단은 더 긴 직사각형) | **1개** 긴 세로 칸만 |
| **우측 광고** | **3개 적층**, **상단 매우 긴 스카이스크래퍼** + **하단 가로 소형 2개** | **1개** 긴 세로 칸만 |
| **빈 슬롯** | 레이아웃 유지용 **빈 영역** 처리 | `src=""`로 **깨진 이미지** 발생 가능 |
| **전체 톤** | **회색 바깥 + 흰색 본문** 구분 | 대부분 흰색 위주 |

## 이번 수정 방향 (참고 페이지에 맞춤)
1. 헤더: 가로 배너 **최대 너비·높이 확대**, 그리드로 로고 중앙·양옆 균형.
2. 좌측: **4칸 스택**, 우측: **3칸 스택**(1칸은 스카이스크래퍼 비율).
3. 빈 칸은 **플레이스홀더**(깨진 이미지 없음).
4. `body`/래퍼에 **연한 회색 배경**, `.page-center`에 **흰 배경·그림자**로 포털형 구분.
5. API `sideLeftStack`(4), `sideRightStack`(3) 추가, 기존 `sideLeft`/`sideRight`는 **1번 칸**과 호환.

---

## 적용 내역 (코드 반영 후)

- **헤더**: 가로 배너 `max-width` 280px / `max-height` 80px, 그리드 `minmax(200px,1fr) auto minmax(200px,1fr)`, 배너 영역에 연한 배경·테두리.
- **본문 래퍼**: `body` 배경 `#e8e8e8`, `.page-center` 흰 배경 + 그림자로 포털형 구분.
- **좌측 윙**: 단일 칸 → **4칸 스택** (상단 3칸 4:3 비율, 하단 1칸 세로형). 너비 **160px**, 뷰포트 **1320px** 이상에서만 표시.
- **우측 윙**: **3칸 스택** (1번 스카이형 긴 세로, 2·3번 가로형).
- **빈 슬롯**: `img`는 `hidden`, 텍스트 플레이스홀더 표시 → 깨진 이미지 아이콘 방지.
- **API/광고 대시보드**: `sideLeftStack` / `sideRightStack` 저장·편집, 광고 리스트에 7행 반영.
