# 대시보드 기사 클릭·모달 도입 — 원인 분석 및 조치 요약

작성일: 2026-04-01

## 1. 기자 대시보드에서 기사 제목 클릭 시 staff/로그인 화면으로 가던 현상

### 코드 기준 원인

1. **페이지 이동형 링크**  
   이전 구현에서 기사 제목이 `<a href="...">` 로 연결되거나, 작성/상세 페이지로 `window.location` 이 사용되면 브라우저가 **인증이 필요한 다른 경로**(예: `nw-office/login.html?role=...` 또는 잘못된 상대경로)로 이동할 수 있습니다. 특히 호스트·경로가 어긋나면 **SPA/정적 서버의 기본 동작**으로 로그인 또는 인덱스 페이지가 뜹니다.

2. **403과 401을 동일하게 처리**  
   목록/상세 API가 403(본인 기사 아님, 권한 없음)을 반환할 때 프론트에서 이를 **“다시 로그인”** 으로 처리하면, 세션은 살아 있어도 로그인 화면으로 보내는 현상이 납니다. (편집장 대시보드 과거 코드에서 특히 흔함.)

3. **잘못된 상세 API**  
   비게시 기사를 **공개 상세** (`/api/articles/public/:id`) 로만 열려 하면 403/404가 나고, 이어지는 **공용 에러 핸들러**가 로그인으로 리다이렉트하는 패턴도 원인 후보입니다.

### 조치

- `nw-office/reporter.html`: 제목 클릭은 **`data-article-open` + `openArticleModal`** 만 사용하고, 상세는 **`/api/articles/:id` + Bearer 토큰** 으로 조회.
- 인증 만료(**401**)일 때만 로그인 이동. **403** 은 모달/메시지로 “권한 없음” 처리.

## 2. 편집장 대시보드에서 기사 클릭 시 에러 화면(데이터 미로딩)

### 코드 기준 원인

1. **공개 API와 권한 API 혼용**  
   송고·대기 상태 기사는 `public` 엔드포인트로는 조회되지 않습니다. `GET /api/articles/public/:id` 는 **published** 만 허용하므로, 편집장이 송고분을 이 URL로 열면 **403** 또는 **404** 가 납니다.

2. **식별자 불일치**  
   목록 row의 `id`와 상세 요청의 `:id`가 다르게 넘어가면(문자열/숫자, `_id` vs `id`, 잘못된 slug) **404** 가 납니다. 백엔드 저장소는 기본적으로 숫자 `id` 를 쓰며, slug는 레코드에 필드가 있을 때만 매칭됩니다.

3. **응답 본문 가정 오류**  
   상세 API는 **기사 객체를 그대로 JSON** 으로 반환합니다. `{ article: ... }` 를 기대하는 클라이언트는 필드를 읽지 못해 “빈 모달”이 됩니다.

4. **JSON 파싱 실패**  
   HTML 에러 페이지가 오면 파싱 실패 후 모든 오류를 한 메시지로 처리하는 경우가 있습니다.

### 조치

- `nw-office/editor.html`: 제목/행 버튼은 모두 **`openEditorArticleModal` → `fetchArticleDetail(API_READ, token, key)`** 로 통일.
- `nw-office/js/article-dashboard-common.js`: `GET ${apiRead}/articles/:key`, 오류 시 **status별 메시지** 분류(401/403/404/500/손상).
- `backend/db/articles.js`: 가능 시 **slug·비숫자 키**도 `resolveArticleRecord` 로 조회해 상세·수정이 같은 키로 동작하도록 보완.

## 3. 아키텍처 정리(현재)

| 구분 | 기자 | 편집장 |
|------|------|--------|
| 목록 | `GET /api/articles` (본인만) | `GET /api/articles` (전체) |
| 상세 | `GET /api/articles/:id` | 동일 |
| 저장/송고 | `PATCH` (reporter 본인) | `PATCH` (필드만: staff 수정 / `action`: 승인·반려) |
| UI | 모달: 미리보기·수정·저장·임시저장·재송고 | 모달: 검토·수정·저장·승인·반려 |

디버그 로그: URL 쿼리 `nwdebug=1` 또는 `localStorage.nwOfficeDebug = '1'`.

---

## 4. 검증 시나리오(수동)

1. 기자: 임시저장·송고 기사 제목 클릭 → 모달 로드 → 수정·저장·재송고 후 목록 갱신.  
2. 편집장: 송고 기사 클릭 → 모달에 본문 표시 → 저장·승인·반려 후 목록 갱신.  
3. 토큰 삭제 후 상세 요청 → **401만** 로그인 유도.  
4. 존재하지 않는 id → “기사 정보를 찾을 수 없습니다.” (404).  
5. 다른 기자 기사를 기자 계정으로 열기 → 403 메시지, 로그인 이동 없음.
