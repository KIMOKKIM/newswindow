# 단일 직접 원인 (합의)

## 확정한 1차 원인 (코드·데이터로 설명 가능, 재현 가능)

**`author_id`가 비어 있는 공개(게시·송고) 기사는 메인 피드에는 나오지만, 기존 `findByAuthor(userId)`만으로는 기자 대시보드에 잡히지 않아 “메인에서 클릭 가능한데 내 목록에는 없음”이 발생한다.**

- 운영 `GET /api/articles/public/1` 응답에 `author_id` 키가 없고 `author_name` 만 있는 패턴과 부합.
- 모달 403 문구는 **상태가 공개 피드 밖**(draft/rejected 등)일 때의 응답이며, **현재 원격에서 “목록 전체 vs 상세” 일괄 검사 시 불일치는 0건**이었다. 동일 문구는 **브라우저가 가진 구 목록·타임 레이스·구버전 배포** 등에서도 사용자에게 동일하게 보일 수 있음.

## 이번 수정의 방향

1. 공개 허용 범위를 **`isPublicFeedReadableStatus` 단일 함수**로 고정(목록·상세 동일).
2. `findByAuthor`에 **이름 기반 보조 매칭**을 두되, **draft 오독 노출을 막기 위해 `author_id` 누락 + 이름 일치는 “게시·송고”에만 적용**.
3. `GET /public/:id` 에 `Cache-Control: no-store` 로 응답 시점 불일치 완화.

## 편집장 목록까지 “사라짐”에 대해

- 저장소·`all()` 기준으로는 **필터로 숨길 수 없음**. 실제 배포 환경에서 **다른 API 베이스**, **응답 오류**, **JSON 손상** 등을 별도 확인해야 함.
