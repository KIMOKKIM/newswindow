# 클릭한 기사 실체·상태 감사

## 운영 API 자동 점검 (2026-04-05, Render `newswindow-backend.onrender.com`)

1. `GET /api/articles/public/list` 로 내려온 **모든** `id` 에 대해 `GET /api/articles/public/:id` 호출.
2. 결과: **목록 길이와 동일하게 상세 200**, 상세 실패 0건.  
   → **현재 배포 기준**으로는 “목록에 있는 id가 상세에서 403” 패턴은 재현되지 않음.

## 샘플·이관 레코드 이슈 (로컬 `backend/data/articles.json` + 운영 id 1 응답)

- id **1** 유형: `status: published`, `author_name` 있음, **`author_id` 필드 없음**(JSON 상 누락).
- **메인**: `listPublishedForMain()` 는 **게시·송고**만 필터 → 노출됨.
- **공개 상세**: 동일 상태 기준이면 **200**.
- **기자 대시보드(기존)**: `findByAuthor(userId)` 가 **`author_id` 일치**만 보므로 → **목록에서 빠짐**.
- **편집장 대시보드**: `articlesDb.all()` → **계속 보여야 함**(같은 저장소).

## 사용자 증상과의 대응

- 모달 **403 / “공개된 기사만…”** : 저장소의 해당 id `status` 가 **draft/rejected/기타**로 내려간 뒤, **브라우저가 예전 목록을 쥔 채 클릭**했거나, **다른 시점·다른 인스턴스의 파일**을 본 경우 등으로 설명 가능. 현 시점 전체 스캔에서는 불일치 0건.
- **기자 목록에서만 사라짐** + 메인에는 있음: **`author_id` 누락 + 공개 상태** 조합이 가장 설명력 있음.
- **편집장까지 없음**: 코드상 `all()` 필터가 없으므로, **별도 오리진·응답 파싱 오류·실제 파일에서 행 삭제** 등을 의심해야 함(코드만으로는 “전부 필터로 숨김” 불가).
